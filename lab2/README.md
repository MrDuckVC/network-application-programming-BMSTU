## 4. Теория и реализация (Примеры из кода)

### 4.1. Доступ к DOM и обработка событий

**Теория:** Для взаимодействия с HTML-элементами используются методы `getElementById` (поиск по ID) и `querySelectorAll` (поиск по CSS-селектору). Для привязки логики к действиям пользователя применяется событийно-ориентированный подход (Event-Driven Programming). Использование устаревших inline-обработчиков (вида `onclick`) заменено на современный метод `addEventListener`. Весь код инкапсулирован внутри события `DOMContentLoaded`, гарантируя, что скрипт начнет работу только после полного построения DOM-дерева браузером.

**Пример из кода:**

```javascript
document.addEventListener('DOMContentLoaded', () => {
    // Получение доступа к HTML-элементам
    const outputElement = document.getElementById('result');
    const digitButtons = document.querySelectorAll('[id ^= "btn_digit_"]');

    // Настройка обработчиков для цифровых кнопок
    digitButtons.forEach(button => {
        button.addEventListener('click', (event) => {
            onDigitButtonClicked(event.target.textContent);
        });
    });
});
```

### 4.2. Архитектурный паттерн «Стратегия»

**Теория:** Использование классической конструкции `switch-case` для маршрутизации математических операций затрудняет масштабирование кода. Для решения этой проблемы применена структура данных «Словарь» (объектные мапы), что реализует поведенческий паттерн проектирования Стратегия (Strategy). Операции разделены на унарные (мгновенные) и бинарные (ожидающие второго операнда).

**Пример из кода:**

```javascript
// Словарь унарных маневров, включая индивидуальную операцию D2S
const unaryManeuvers = {
    'btn_op_sign': x => x * -1,
    'btn_op_sqrt': x => Math.sqrt(x),
    'btn_op_x2': x => x * x,
    'btn_op_factorial': x => factorial(x),
    'btn_op_d2s': x => x / 1.02749 // Конвертация дней в марсианские солы
};
```

### 4.3. История текущего ввода (Доп. задание)

**Теория:** Для реализации динамической телеметрии ввода (истории) выделена отдельная переменная состояния `expressionHistory`. Обновление экрана происходит при каждом вводе цифры или операции. Игнорирование иных операций, кроме `+` и `-`, выполнено с помощью защитного выражения (Guard Clause).

**Пример из кода:**

```javascript
function updateHistoryDisplay() {
    if (!historyElement) return;

    if (expressionHistory.includes('=')) {
        historyElement.textContent = expressionHistory;
    } else {
        // Учитываем только сложение и вычитание (согласно заданию)
        let isValidOp = selectedOperation === 'btn_op_plus' || selectedOperation === 'btn_op_minus';
        let activeInput = (isValidOp && b !== '') ? b : '';

        historyElement.textContent = expressionHistory + activeInput;
    }
}
```

### 4.4. Управление состоянием и защита от ошибок

**Теория:** Манипуляция стилями производится через безопасное API `classList.toggle` или `setAttribute`, что обеспечивает строгую изоляцию логики от CSS-стилей (например, при смене цветовой темы). Для защиты от критических математических сбоев используется `try...catch` и проверка на `NaN` или переполнение.

**Пример из кода:**

```javascript
function factorial(num) {
    num = Math.trunc(num);
    if (num < 0) return NaN; // Инженерная защита от падения скрипта
    if (num === 0 || num === 1) return 1;
    let result = num;
    while (num > 1) {
        num--;
        result *= num;
    }
    return result;
}
```
