## 4. Теория и реализация (Примеры из кода)

### 4.1. Архитектура SPA и Компонентный подход

**Теория:** В отличие от Multi-Page Application (Лабораторная №1), SPA не перезагружает страницу при навигации. Маршрутизация осуществляется путем очистки корневого элемента (`innerHTML = ''`) и динамической вставки HTML-кода нужной страницы через метод `insertAdjacentHTML`. Для устранения дублирования кода интерфейс разбит на классы-компоненты.

**Пример из кода (Инициализация и Роутинг):**

```javascript
// pages/main/index.js
clickCard(id) {
    // При клике на кнопку "Подробнее" создаем и рендерим страницу продукта
    const productPage = new ProductPage(this.parent, id);
    productPage.render();
}

renderCards(dataArray) {
    const container = document.getElementById('carousel-inner-container');
    container.innerHTML = ''; // Очистка контейнера

    dataArray.forEach((item, index) => {
        const productCard = new ProductCardComponent(container);
        // Проброс callback-функции для сохранения контекста вызова
        productCard.render(item, index, () => this.clickCard(item.id));
    });
}
```

### 4.2. Полноэкранная карусель и Layout (Доп. задание)

**Теория:** Для создания полноэкранного интерфейса без жесткого указания высоты в пикселях была применена математика Flexbox. Тегу `<body>` задан класс `min-vh-100` и `d-flex flex-column`. Тег `<main>` получил `flex-grow-1`, что заставило его занять всё свободное пространство между хедером и футером.
Чтобы обойти проблему схлопывания процентной высоты (`h-100`) внутри Flex-контейнеров, слайдам карусели было задано абсолютное позиционирование.

**Пример из кода (Сложная сетка Bootstrap внутри слайда):**

```html
<img src="${data.carouselSrc}" class="position-absolute top-0 start-0 w-100 h-100 object-fit-cover z-0" alt="SpaceX">

<div class="container-fluid position-relative z-1 h-100 d-flex flex-column justify-content-end pb-5">
    <div class="row w-100 mb-5">
        <div class="col-10 offset-1 col-md-5 offset-md-1 col-lg-3 offset-lg-2 pe-auto">
            <h2 class="text-white text-uppercase fw-bold mb-3">${data.spacexTitle}</h2>
            <p class="text-light mb-4">${data.spacexText}</p>
            <button class="btn btn-outline-light px-4 py-2 text-uppercase" id="click-card-${data.id}">Подробнее</button>
        </div>
    </div>
</div>
```

### 4.3. Алгоритмическая панель (ДЗ №1)

**Теория:** Стандартные боковые кнопки управления Bootstrap Carousel имеют невидимую зону клика шириной 15% от краев экрана. Чтобы панель с алгоритмами была кликабельной в правом верхнем углу, применено вычисление `right: 15%`, выводящее блок из "мертвой зоны" перелистывания. Логика выполнения алгоритмов изолирована с помощью блочной области видимости `{ }` внутри конструкции `switch`.

**Пример из кода (Задача 3.2: Инверсия массива с указателями и do...while):**

```javascript
case 4: {
    const payload = ["Панели", "Ровер", "Провизия", "Медикаменты", "Кислород"];
    const num = 2; // Оставляем первые 2 элемента на месте
    let arr = [...payload];
    let start = 0;
    let end = arr.length - 1;

    // Сдвиг указателя в зависимости от знака числа
    if (num > 0) start = num;
    else if (num < 0) end = arr.length - 1 + num;

    // Цикл с постусловием (требование ДЗ)
    if (start < end) {
        do {
            let temp = arr[start];
            arr[start] = arr[end];
            arr[end] = temp;
            start++;
            end--;
        } while (start < end);
    }
    // Вывод в DOM...
    break;
}
```
